# 스프린트 3 테스트 시나리오

스프린트 3의 목표는 아래 3가지를 검증하는 것이다.
- 저장 직후 흐름이 더 자연스러워졌는지
- 후기 작성 경험이 방문 완료 상태와 잘 연결되는지
- 카드와 상세 페이지가 후속 행동을 유도하는 데 도움이 되는지

## 테스트 범위

- 새 릴스 저장 후 상세 페이지로 이동하는 흐름
- 후기 없는 릴스 상세에서 후기 작성 CTA 노출
- 후기 작성 시 방문 완료 자동 반영
- 홈 카드의 `방문 완료 / 후기 있음` 배지 표시
- 여러 릴스를 연속 저장할 때 현재 흐름이 불편하지 않은지

## 사전 조건

- 개발 서버가 실행 중이어야 한다
- 테스트 계정으로 로그인 가능해야 한다
- 테스트용 릴스를 새로 저장할 수 있어야 한다

## 테스트 시나리오 1

이름
- 새 릴스 저장 후 상세 이동 흐름 확인

목적
- 저장 직후 상세 페이지로 이동하는 흐름이 자연스러운지 확인한다

사전 조건
- 로그인된 상태여야 한다

실행 순서
1. 홈에서 `+ 릴스 추가`를 누른다
2. 릴스 URL을 입력한다
3. 카테고리, 태그, 메모는 입력하고 후기는 비워둔다
4. 저장한다
5. 저장 후 이동한 화면을 확인한다

기대 결과
- 홈이 아니라 해당 릴스의 상세 페이지로 이동한다
- 저장 완료 메시지가 노출된다
- 저장 직후 다음 행동이 무엇인지 이해할 수 있다

체감 포인트
- 홈으로 돌아가는 것보다 더 자연스러운가
- 저장 후 바로 상세를 보는 흐름이 도움이 되는가
- 반대로 바로 다음 릴스를 추가하고 싶은데 흐름이 끊기는 느낌이 있는가

## 테스트 시나리오 2

이름
- 후기 없는 상세 페이지에서 행동 유도 확인

목적
- 후기 없는 릴스 상세 페이지가 후속 행동을 자연스럽게 유도하는지 확인한다

사전 조건
- 후기가 없는 릴스가 하나 있어야 한다

실행 순서
1. 후기 없이 저장한 릴스 상세 페이지를 연다
2. 상단 상태 배지를 확인한다
3. `아직 후기가 없습니다` 영역과 `후기 작성하러 가기` 링크를 확인한다

기대 결과
- 상태 배지에 `미방문`, `후기 없음`이 보인다
- 후기 작성 CTA가 노출된다

체감 포인트
- 안내 문구가 과하지 않은가
- 다음 행동을 유도하는 방식이 자연스러운가
- 나중에 쓰고 싶을 때 부담스럽지 않은가

## 테스트 시나리오 3

이름
- 후기 작성 시 방문 완료 자동 반영 확인

목적
- 후기 작성과 방문 완료 상태가 자연스럽게 연결되는지 확인한다

사전 조건
- 후기가 없는 릴스가 하나 있어야 한다

실행 순서
1. 상세 페이지에서 `후기 작성하러 가기`를 누른다
2. 후기 내용을 입력한다
3. 저장한다
4. 저장 후 다시 상세 페이지 상태를 확인한다

기대 결과
- 수정 완료 메시지가 노출된다
- 상태가 `방문 완료`, `후기 있음`으로 바뀐다

체감 포인트
- 후기를 쓰면 방문 완료로 바뀌는 것이 자연스러운가
- 자동 반영이 예상 가능한 동작처럼 느껴지는가

## 테스트 시나리오 4

이름
- 홈 카드 상태 배지 확인

목적
- 홈 카드에서 상태 배지가 재탐색에 실제로 도움이 되는지 확인한다

사전 조건
- 후기 있음 또는 방문 완료 상태의 릴스가 1개 이상 있어야 한다

실행 순서
1. 홈으로 이동한다
2. 방금 수정한 릴스 카드를 찾는다
3. 카드 내 상태 배지를 확인한다

기대 결과
- `방문 완료`, `후기 있음` 배지가 카드에 노출된다

체감 포인트
- 배지가 실제로 도움이 되는가
- 카드가 복잡해지지는 않았는가
- 태그, 카테고리, 메모와 함께 봤을 때 정보량이 적절한가

## 테스트 시나리오 5

이름
- 연속 저장 흐름 점검

목적
- 여러 릴스를 연속으로 저장하는 사용자 관점에서 현재 흐름이 불편하지 않은지 확인한다

사전 조건
- 새로 저장할 릴스가 하나 더 있어야 한다

실행 순서
1. 새 릴스를 하나 더 저장한다고 가정하고 `+ 릴스 추가`로 이동한다
2. 새 릴스를 저장한다
3. 다시 상세 페이지로 이동하는 흐름을 체감한다

기대 결과
- 기본 흐름 자체는 문제 없이 동작한다

체감 포인트
- 연속 저장할 때도 지금 방식이 괜찮은가
- `계속 추가` 버튼이 필요하다고 느껴지는가
