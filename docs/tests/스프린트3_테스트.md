# 스프린트 3 테스트 시나리오

스프린트 3의 목표는 아래 3가지를 검증하는 것이다.
- 저장 직후 흐름이 더 자연스러워졌는지
- 후기 작성 경험이 방문 완료 상태와 잘 연결되는지
- 카드와 상세 페이지가 후속 행동을 유도하는 데 도움이 되는지

## 테스트 범위

- 새 릴스 저장 후 홈으로 돌아가는 흐름
- 후기 없는 릴스 상세에서 후기 작성 CTA 노출
- 후기 작성 시 방문 완료 자동 반영
- 상세 썸네일의 별 아이콘으로 방문 상태가 충분히 전달되는지
- 여러 릴스를 연속 저장할 때 `저장하고 계속 추가` 흐름이 필요한지

## 사전 조건

- 개발 서버가 실행 중이어야 한다
- 테스트 계정으로 로그인 가능해야 한다
- 테스트용 릴스를 새로 저장할 수 있어야 한다

## 테스트 시나리오 1

이름
- 새 릴스 저장 후 홈 복귀 흐름 확인

목적
- 저장 직후 홈으로 돌아가는 흐름이 아카이빙 사용 패턴에 더 자연스러운지 확인한다

사전 조건
- 로그인된 상태여야 한다

실행 순서
1. 홈에서 `+ 릴스 추가`를 누른다
2. 릴스 URL을 입력한다
3. 카테고리, 태그, 메모는 입력하고 후기는 비워둔다
4. 저장한다
5. 저장 후 이동한 화면을 확인한다

기대 결과
- 저장 후 홈으로 돌아간다
- 다음 릴스를 이어서 저장하거나 목록을 계속 탐색하기 쉬워야 한다

체감 포인트
- 아카이빙 앱 관점에서 홈 복귀가 더 자연스러운가
- 저장 후 바로 다음 릴스를 추가하기 편한가
- 저장 완료 메시지가 꼭 필요한가

## 테스트 시나리오 2

이름
- 후기 없는 상세 페이지에서 행동 유도 확인

목적
- 후기 없는 릴스 상세 페이지가 후속 행동을 자연스럽게 유도하는지 확인한다

사전 조건
- 후기가 없는 릴스가 하나 있어야 한다

실행 순서
1. 후기 없이 저장한 릴스 상세 페이지를 연다
2. 썸네일 위 별 아이콘 상태를 확인한다
3. `아직 후기가 없습니다` 영역과 `후기 작성하러 가기` 링크를 확인한다

기대 결과
- 후기 작성 CTA가 노출된다
- 별 아이콘만으로 현재 방문 상태를 이해할 수 있어야 한다

체감 포인트
- 안내 문구가 과하지 않은가
- 다음 행동을 유도하는 방식이 자연스러운가
- 나중에 쓰고 싶을 때 부담스럽지 않은가
- `미방문`, `후기 없음` 배지가 없어도 충분한가

## 테스트 시나리오 3

이름
- 후기 작성 시 방문 완료 자동 반영 확인

목적
- 후기 작성과 방문 완료 상태가 자연스럽게 연결되는지 확인한다

사전 조건
- 후기가 없는 릴스가 하나 있어야 한다

실행 순서
1. 상세 페이지에서 `후기 작성하러 가기`를 누른다
2. 후기 내용을 입력한다
3. 저장한다
4. 저장 후 다시 상세 페이지 상태를 확인한다

기대 결과
- 상세 썸네일의 별 아이콘이 방문 완료 상태로 바뀐다
- 후기 내용이 저장된다

체감 포인트
- 후기를 쓰면 방문 완료로 바뀌는 것이 자연스러운가
- 자동 반영이 예상 가능한 동작처럼 느껴지는가
- `후기 작성하러 가기`를 눌렀을 때 후기 입력 영역으로 바로 이동하는 것이 자연스러운가

## 테스트 시나리오 4

이름
- 홈 카드 정보량 확인

목적
- 홈 카드에 상태 배지가 없어도 정보량이 더 적절한지 확인한다

사전 조건
- 후기 있음 또는 방문 완료 상태의 릴스가 1개 이상 있어야 한다

실행 순서
1. 홈으로 이동한다
2. 방금 수정한 릴스 카드를 찾는다
3. 카드에 표시되는 정보량을 확인한다

기대 결과
- 카드에 불필요한 상태 배지가 보이지 않아야 한다

체감 포인트
- 별 아이콘만으로 방문 완료를 충분히 인식할 수 있는가
- 카드가 복잡해지지는 않았는가
- 태그, 카테고리, 메모와 함께 봤을 때 정보량이 적절한가

## 테스트 시나리오 5

이름
- 연속 저장 흐름 점검

목적
- 여러 릴스를 연속으로 저장하는 사용자 관점에서 현재 흐름이 불편하지 않은지 확인한다

사전 조건
- 새로 저장할 릴스가 하나 더 있어야 한다

실행 순서
1. 새 릴스를 하나 더 저장한다고 가정하고 `+ 릴스 추가`로 이동한다
2. 새 릴스를 저장한다
3. `저장하고 계속 추가` 버튼이 필요한지 또는 이미 유용한지 확인한다

기대 결과
- 기본 흐름 자체는 문제 없이 동작한다
- 연속 저장을 위한 선택지가 있어야 한다

체감 포인트
- 연속 저장할 때도 지금 방식이 괜찮은가
- `저장하고 계속 추가` 버튼이 실제로 도움이 되는가

- 시나리오 5: 성공

판단 질문
- 저장 후 홈 복귀가 더 자연스럽다
- 후기 작성 CTA는 유용하다
- 후기 작성 시 자동 방문 완료는 자연스럽다
- 카드 배지는 없어도 된다
- `저장하고 계속 추가` 버튼이 필요하다

관찰 메모
- 저장 완료 메시지는 없어도 된다고 판단
- 상세 페이지의 `미방문`, `후기 없음`, `후기 있음` 배지는 제거하는 쪽이 더 자연스러움
- 방문 상태는 썸네일 위 별 아이콘만으로 충분히 전달 가능
- 후기 작성 CTA는 유지하되, 후기 입력 영역으로 바로 이동하는 것이 자연스러움
- 테스트 중 인스타 썸네일 추출 실패 및 인스타 CDN 이미지 `403` 반복 요청 사례가 확인됨
- 이 문제는 스프린트 3 차단 이슈는 아니지만, 스프린트 5에서 fallback 정책과 만료 대응 기준을 정리할 필요가 있음
