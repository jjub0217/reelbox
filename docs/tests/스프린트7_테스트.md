# 스프린트 7 테스트 시나리오

스프린트 7의 목표는 실제 사용자 테스트에서 발견된 핵심 불편 포인트를 보정하는 것이다.

## 테스트 범위

- 수정 후 메인 목록 복귀 시 이전 위치 복원
- 썸네일 자동 추출 품질 개선 방향 검토
- 자동 추출 실패 시 수동 썸네일 지정 필요성 확인

## 사전 조건

- 개발 서버 또는 배포 환경에 접속 가능해야 한다
- 테스트 계정으로 로그인 가능해야 한다
- 메인 목록에서 충분히 아래쪽에 있는 릴스를 대상으로 테스트할 수 있어야 한다

## 테스트 시나리오 1

이름
- 수정 후 메인 목록 위치 복원 확인

목적
- 수정 후 상세 페이지를 거쳐 메인으로 돌아갈 때, 이전에 보던 릴스 위치로 복귀하는지 확인한다

사전 조건
- 메인 목록에 여러 릴스가 있어 스크롤이 충분히 내려가야 한다

실행 순서
1. 메인 목록을 아래로 스크롤한다
2. 특정 릴스를 클릭한다
3. 수정 화면으로 이동해 일부 내용을 수정한다
4. 수정 완료 후 상세 페이지로 이동한다
5. 상단 뒤로가기 아이콘을 누른다

기대 결과
- 메인 목록 상단이 아니라, 방금 보고 있던 릴스 위치 근처로 돌아가야 한다

현재 상태
- 완료로 간주

관찰 메모
- 사용자 테스트를 여러 차례 반복한 결과 해결된 것으로 판단

## 테스트 시나리오 2

이름
- 썸네일 자동 추출 체감 품질 확인

목적
- 실제 사용 시 썸네일 자동 추출 성공률이 충분한지 확인한다

사전 조건
- 여러 종류의 릴스 URL을 준비할 수 있어야 한다

실행 순서
1. 서로 다른 인스타 릴스/게시물 URL을 여러 개 저장한다
2. 저장 직전 미리보기의 썸네일 상태를 확인한다
3. 저장 후 메인 카드에서 썸네일 상태를 확인한다
4. 상세 페이지에서도 같은 항목의 썸네일 상태를 확인한다

기대 결과
- 체감상 대부분의 릴스에서 썸네일이 정상적으로 보이는지 판단할 수 있어야 한다
- 썸네일이 없는 항목 비율이 여전히 높다면 다음 개선이 필요하다

체감 포인트
- 실제 사용 기준으로 썸네일 성공률이 충분한가
- fallback만으로 용인 가능한 수준인가

## 테스트 시나리오 3

이름
- 썸네일 실패 시 수동 지정 필요성 확인

목적
- 자동 추출만으로 충분한지, 아니면 사용자가 직접 썸네일을 보완할 기능이 필요한지 확인한다

사전 조건
- 썸네일이 실패한 릴스가 1개 이상 있어야 한다

실행 순서
1. 썸네일이 없는 릴스를 목록에서 확인한다
2. 해당 릴스를 다시 찾을 때 불편한지 체감한다
3. 그 상황에서 직접 이미지를 넣을 수 있으면 도움이 될지 판단한다

기대 결과
- 수동 썸네일 지정 기능의 필요성을 실제 사용 기준으로 판단할 수 있어야 한다

체감 포인트
- 썸네일이 없으면 다시 찾기 어려운가
- 자동 추출만으로 충분한가
- 수동 업로드가 있어야 한다고 느껴지는가
