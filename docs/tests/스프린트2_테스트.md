# 스프린트 2 테스트 시나리오

스프린트 2의 목표는 아래 3가지를 검증하는 것이다.
- 홈 화면이 단순하게 유지되는지
- 상태 기반 탐색이 별도 페이지에서 자연스럽게 동작하는지
- `방문 완료 / 후기 있음` 탭과 검색이 함께 동작하는지

## 테스트 범위

- 홈에서 상태 필터 UI가 제거되었는지
- 사이드바에서 `기록 보기` 메뉴로 진입할 수 있는지
- `기록 보기` 페이지에서 `방문 완료 / 후기 있음` 탭 전환이 되는지
- 각 탭에서 검색이 정상 동작하는지
- 결과가 없을 때 빈 상태 문구가 맞게 보이는지

## 사전 조건

- 개발 서버가 실행 중이어야 한다
- 테스트 계정으로 로그인 가능해야 한다
- 아래 조건을 만족하는 테스트 데이터가 준비되어 있으면 좋다
- `visited = true` 인 릴스가 1개 이상
- `review` 값이 있는 릴스가 1개 이상
- 방문 완료이지만 후기가 없는 릴스가 있으면 탭 구분 테스트에 유리하다

## 테스트 시나리오 1

이름
- 홈 화면 단순화 확인

목적
- 홈에서 상태 필터 UI가 제거되고, 검색 + 카테고리 중심으로 보이는지 확인한다

사전 조건
- 로그인된 상태여야 한다

실행 순서
1. 홈 화면으로 이동한다
2. 검색창 아래 영역을 확인한다

기대 결과
- 홈에는 기존 카테고리 필터만 보인다
- `방문 완료`, `미방문`, `후기 있음`, 정렬 셀렉트 같은 상태 UI는 홈에서 보이지 않는다

## 테스트 시나리오 2

이름
- 사이드바 `기록 보기` 진입

목적
- 세부 상태 탐색 기능이 사이드바 메뉴에서 별도 페이지로 이동하는지 확인한다

사전 조건
- 로그인된 상태여야 한다

실행 순서
1. 홈 화면 우측 상단 햄버거 버튼을 누른다
2. 사이드바 메뉴에서 `기록 보기`를 클릭한다

기대 결과
- `/archive` 페이지로 이동한다
- 상단에 `기록 보기` 제목이 보인다
- `방문 완료`, `후기 있음` 탭이 보인다

## 테스트 시나리오 3

이름
- `방문 완료` 탭 목록 확인

목적
- `방문 완료` 탭에서 방문 완료 처리된 릴스만 노출되는지 확인한다

사전 조건
- `visited = true` 인 릴스가 1개 이상 있어야 한다

실행 순서
1. `기록 보기` 페이지로 이동한다
2. 기본 탭이 `방문 완료`인지 확인한다
3. 목록에 노출되는 릴스를 확인한다

기대 결과
- 방문 완료 처리된 릴스가 보인다
- 방문 완료되지 않은 릴스는 보이지 않아야 한다

## 테스트 시나리오 4

이름
- `후기 있음` 탭 목록 확인

목적
- 후기 값이 있는 릴스만 별도로 모아서 볼 수 있는지 확인한다

사전 조건
- `review` 값이 있는 릴스가 1개 이상 있어야 한다

실행 순서
1. `기록 보기` 페이지에서 `후기 있음` 탭을 클릭한다
2. 목록에 노출되는 릴스를 확인한다

기대 결과
- 후기 값이 있는 릴스만 보여야 한다
- 후기 값이 없는 릴스는 보이지 않아야 한다

비고
- 후기 값이 있는 릴스가 동시에 방문 완료일 수도 있다

## 테스트 시나리오 5

이름
- 탭 내 검색 동작 확인

목적
- `기록 보기` 페이지 안에서도 현재 탭 조건을 유지한 채 검색이 동작하는지 확인한다

사전 조건
- `방문 완료` 또는 `후기 있음` 대상 릴스 중 검색 가능한 메모/후기/태그/카테고리 데이터가 있어야 한다

실행 순서
1. `기록 보기` 페이지에서 `후기 있음` 탭을 선택한다
2. 검색창에 특정 후기 키워드를 입력한다
3. 결과를 확인한다
4. 다시 `방문 완료` 탭으로 이동한다
5. 검색어가 유지되는지 확인한다

기대 결과
- 현재 탭 조건을 유지한 채 검색 결과가 필터링된다
- URL에 `tab`과 `search`가 함께 반영된다
- 탭을 바꿔도 검색 동작이 유지된다

## 테스트 시나리오 6

이름
- 빈 결과 상태 확인

목적
- 탭 또는 검색 조건에 맞는 릴스가 없을 때 안내 문구가 적절한지 확인한다

사전 조건
- 검색 결과가 없도록 만들 수 있어야 한다

실행 순서
1. `기록 보기` 페이지에서 아무 탭이나 선택한다
2. 존재하지 않는 키워드를 검색한다

기대 결과
- `조건에 맞는 릴스가 없습니다` 문구가 표시된다
