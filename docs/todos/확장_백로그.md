# ReelBox 확장 백로그

이 문서는 스프린트 1~5 완료 이후, 다음 단계에서 검토할 확장 기능 후보를 정리한 문서다.

기준 원칙
- 개인 아카이브의 핵심 가치와 직접 연결되는지 먼저 본다
- 지금 당장 구현하지 않아도 되는 기능은 명확히 뒤로 미룬다
- 재미있는 기능보다 자주 다시 찾고 활용하게 만드는 기능을 우선한다

## 현재 제품 기준

지금 ReelBox는 아래 흐름까지는 잘 정리된 상태다.
- 릴스 저장
- 태그/카테고리/메모/후기 정리
- 검색과 기록 보기
- 후기 작성과 방문 상태 관리

다음 단계부터는 단순 저장이 아니라, 저장한 데이터를 얼마나 오래 활용하게 만들지에 집중해야 한다.

## 우선 검토 대상

### 1. 내보내기 기능

왜 중요한가
- 개인 아카이브 서비스에서는 데이터 소유감이 중요하다
- 사용자는 언제든 내 데이터를 밖으로 꺼낼 수 있다고 느껴야 안심한다
- 다른 도구로 옮기거나 백업할 수 있어야 장기 보관 가치가 생긴다

검토 포인트
- CSV export가 먼저인지, JSON export가 먼저인지
- 어떤 필드를 내보낼지
  - URL
  - 카테고리
  - 태그
  - 메모
  - 후기
  - 방문 여부
  - 생성일
- 내보내기 위치
  - 사이드바 메뉴
  - 설정 페이지
  - 관리자와 분리된 사용자 메뉴

장점
- 개인 아카이브 서비스와 직접 연결됨
- 구현 난이도 대비 체감 가치가 큼
- 백업 기능 역할도 함께 수행 가능

주의할 점
- 파일 포맷을 잘못 잡으면 나중에 다시 바꾸기 어려움
- 첫 버전은 최대한 단순해야 함

현재 추천 우선순위
- 높음

### 2. 후기 구조화

왜 중요한가
- 지금 후기는 자유 텍스트라서 검색은 되지만 비교와 정리가 어렵다
- 구조화 필드가 있으면 나중에 다시 찾을 때 더 유용해진다

후보 필드
- 별점
- 추천 여부
- 재방문 의사
- 가격 만족도

장점
- 후기의 활용도가 높아짐
- 필터와 검색 확장 가능성이 커짐
- 개인 기록이 더 축적형 데이터가 됨

주의할 점
- 너무 많은 필드를 넣으면 후기 작성이 오히려 귀찮아질 수 있음
- 첫 버전은 1~2개 정도만 선택해야 함

현재 추천 우선순위
- 중간

### 3. 리마인드 기능

왜 중요한가
- 저장만 하고 잊는 문제를 줄일 수 있다
- 미방문 릴스를 다시 행동으로 이어지게 만들 수 있다

후보 시나리오
- 오래된 미방문 릴스 다시 보기
- 후기 없는 방문 완료 릴스 다시 보기
- 주간 정리 화면

장점
- 저장 이후 활용도를 높일 수 있음
- 서비스 재방문 이유를 만들 수 있음

주의할 점
- 알림/리마인드는 구현 비용이 커질 수 있음
- 지금 단계에서는 과할 가능성이 있음
- 실제로 사용자에게 도움이 되는 타이밍을 잡기 어려움

현재 추천 우선순위
- 낮음

## 추가 후보

### 4. 태그 병합

설명
- 오탈자나 비슷한 의미의 태그를 하나로 합치는 기능

필요한 이유
- 태그 관리가 깊어지면 수정만으로는 한계가 생김

현재 추천 우선순위
- 중간 이하

### 5. 고급 검색

설명
- 카테고리 + 태그 + 후기 있음 + 방문 완료 등을 조합해서 찾는 기능

필요한 이유
- 데이터가 많아질수록 유용함

현재 추천 우선순위
- 중간 이하

### 6. 공유 또는 외부 링크화

설명
- 특정 릴스 기록을 외부에 공유하거나, 읽기 전용 링크로 보여주는 기능

현재 추천 우선순위
- 낮음

이유
- 현재 제품은 개인 아카이브 성격이 더 강함

## 지금 당장 만들지 않을 기능

- 소셜 기능
- 공개 피드
- 사용자 간 공유 기반 기능
- 관리자 기능 고도화

이유
- 지금 제품의 중심은 개인 아카이브다
- 핵심 사용자 가치와 거리가 있다

## 추천 순서

### 1차 추천
- 내보내기 기능

이유
- 가장 개인 아카이브답다
- 구현 대비 체감 가치가 높다

### 2차 추천
- 후기 구조화

이유
- 검색성과 기록 활용도를 높여준다

### 3차 추천
- 리마인드 기능

이유
- 매력적이지만 지금 단계에서는 상대적으로 무겁다

## 논의 질문

- 사용자가 가장 먼저 원할 만한 확장 기능은 무엇인가
- “내 데이터는 언제든 꺼낼 수 있다”는 느낌이 지금 필요한가
- 후기 입력을 더 구조화해도 사용성이 유지될까
- 리마인드는 실제 문제를 해결할까, 아니면 기능만 늘릴까
