# ReelBox 어드민 확장 백로그

이 문서는 개인 아카이빙 앱인 ReelBox에서, 사용자 기능 확장보다 운영과 관리 측면에서 검토할 어드민 기능 후보를 정리한 문서다.

기준 원칙
- 사용자 경험을 해치지 않고 운영 판단에 도움 되는 기능을 우선한다
- 개인 아카이빙 앱 성격에 맞지 않는 과한 관리 기능은 뒤로 미룬다
- “누가 뭘 많이 쓰는가”보다 “어디서 서비스가 깨지는가”를 먼저 본다

## 현재 어드민 기능 상태

현재 어드민 영역에는 아래 기능이 이미 있다.
- 관리자 대시보드
- 회원 목록
- 탈퇴 로그 조회
- 기본 통계 차트

즉 완전히 비어 있는 상태는 아니고, 다음 단계에서는 운영 판단에 더 직접 도움이 되는 정보를 보강하는 것이 적절하다.

## 우선 검토 대상

### 1. 운영 품질 모니터링

왜 중요한가
- 지금까지 실제 테스트에서도 썸네일 추출 실패, 저장 후 썸네일 불일치 같은 운영 이슈가 확인됐다
- 개인 서비스라도 운영자가 어디서 문제가 생기는지 빨리 볼 수 있어야 한다

후보 기능
- 썸네일 없는 릴스 개수
- 최근 썸네일 추출 실패 사례 목록
- 저장/수정 실패 건수 추적
- 최근 에러 유형 요약

예시 지표
- 썸네일 없는 릴스 수
- 최근 7일 신규 저장 중 썸네일 없는 비율
- 실패 URL 패턴

현재 추천 우선순위
- 매우 높음

### 2. 탈퇴/이탈 분석 강화

왜 중요한가
- 이미 탈퇴 사유는 수집하고 있으므로, 그 데이터를 더 잘 읽는 쪽이 효율적이다
- 지금 상태에서는 “몇 명이 탈퇴했는가” 수준이고, 왜/언제 떠나는지 해석은 약하다

후보 기능
- 기간별 탈퇴 추이 세분화
- 가입 후 탈퇴까지 걸린 기간 분포
- 탈퇴 사유 조합 보기
- 최근 가입자 대비 이탈률 보기

현재 추천 우선순위
- 높음

### 3. 회원 관리 강화

왜 중요한가
- 문의 대응이나 운영 점검 시 특정 사용자의 상태를 빨리 확인할 수 있어야 한다

후보 기능
- 회원 검색 조건 확대
- 가입일 기준 정렬
- 최근 활동일 또는 최근 저장일 표시
- 사용자별 저장 패턴 요약

현재 추천 우선순위
- 높음

### 4. 데이터 품질 관리

왜 중요한가
- 개인 아카이빙 앱이라도 데이터가 쌓이면 품질 이슈가 생긴다
- 운영자는 시스템 전체에서 어떤 형태의 데이터가 깨지는지 볼 수 있어야 한다

후보 기능
- 썸네일 없는 릴스 목록
- 후기 없는 저장 비율
- 카테고리 없는 릴스 비율
- 사용자별 태그/카테고리 과다 사용 여부 확인

현재 추천 우선순위
- 중간

### 5. 관리자 액션 로그

왜 중요한가
- 관리자가 늘어나거나 운영 행위가 많아지면 기록이 필요하다

후보 기능
- 관리자 권한 부여/해제 로그
- 회원 상태 조회 로그
- 운영자가 수행한 주요 변경 이력

현재 추천 우선순위
- 중간 이하

이유
- 현재 단계에서는 운영자 수가 많지 않고, 먼저 필요한 건 로그보다 가시성이다

## 구체적인 기능 후보

### A. 썸네일 품질 보드

설명
- 썸네일 관련 문제를 별도로 모아서 보는 운영 화면

포함 후보
- 썸네일 없음
- 기본 fallback 사용 중
- 최근 실패 URL
- 인스타 경로별 실패 사례

추천 이유
- 최근 실제 문제와 가장 직접 연결된다

### B. 사용자 활동 요약

설명
- 사용자별 저장/방문/후기 활동을 간단히 보여주는 화면

포함 후보
- 총 릴스 수
- 방문 완료 수
- 후기 작성 수
- 최근 저장일

추천 이유
- 개인 앱이라도 실제 사용성을 간접적으로 볼 수 있다

### C. 탈퇴 분석 보드

설명
- 탈퇴 이유와 시점을 더 입체적으로 읽는 화면

포함 후보
- 사유별 추이
- 최근 30일 탈퇴율
- 가입 후 빠른 이탈 사용자 비중

추천 이유
- 운영 판단에 직접적이다

## 지금 당장 만들지 않을 것

- 관리자용 과도한 권한 체계
- 사용자 제재 시스템
- 공개 피드 관리 기능
- 콘텐츠 추천 운영 기능

이유
- ReelBox는 개인 아카이빙 앱이고, 대규모 커뮤니티 운영 도구가 아니다

## 추천 순서

### 1차 추천
- 운영 품질 모니터링

이유
- 지금까지 실제로 발생한 문제를 가장 빨리 줄일 수 있다

### 2차 추천
- 탈퇴/이탈 분석 강화

이유
- 이미 쌓이고 있는 데이터를 더 잘 읽게 해준다

### 3차 추천
- 회원 관리 강화

이유
- 운영 대응 속도를 높일 수 있다

### 4차 추천
- 데이터 품질 관리

이유
- 장기적으로 필요하지만 지금 당장은 1~3보다 급하지 않다

## 논의 질문

- 운영자가 가장 자주 확인해야 하는 정보는 무엇인가
- 지금 어드민에서 가장 답답한 지점은 어디인가
- “회원 수”보다 더 중요한 운영 지표는 무엇인가
- 썸네일/저장 실패 같은 품질 이슈를 따로 보드로 분리할 가치가 있는가
