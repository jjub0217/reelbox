# ReelBox 사용자 테스트 이슈

이 문서는 실제 배포 환경에서 사용자가 직접 사용해보며 발견한 불편 포인트를 정리한 문서다.

기준
- 코드 의도와 실제 사용자 경험이 어긋나는 지점을 먼저 기록한다
- 단순 취향 차이보다, 사용 흐름이 끊기거나 결과 품질이 떨어지는 문제를 우선한다
- 각 항목은 다음 스프린트 후보로 연결할 수 있을 정도로 구체적으로 적는다

## 현재 기록된 이슈

### 1. 수정 후 메인으로 돌아갈 때 스크롤 위치가 유지되지 않음

상황
- 사용자가 메인 목록을 스크롤하다가 특정 릴스를 선택한다
- 릴스를 수정한 뒤 상세 페이지로 이동한다
- 상세 페이지에서 뒤로가기 아이콘을 눌러 메인으로 돌아간다

현재 동작
- 메인페이지 상단으로 이동된다

사용자 기대
- 방금 수정했던 릴스가 있던 위치로 돌아가야 한다

문제 성격
- UX 이슈

영향
- 탐색 맥락이 끊긴다
- 긴 목록을 다시 스크롤해야 해서 피로감이 커진다
- 아카이빙 앱 특성상 반복 탐색 경험이 나빠진다

우선순위
- 높음

개선 방향 후보
- 메인 목록 스크롤 위치 저장/복원
- 상세 페이지 뒤로가기를 브라우저 히스토리 기반으로 처리
- 수정 후 상세 이동 흐름 자체를 재검토할지 여부 확인

### 2. 썸네일 자동 추출 실패율이 높음

상황
- 실제 사용 중 썸네일을 불러오지 못한 릴스 비율이 높게 체감됨
- 대략 절반 수준까지 느껴질 정도로 빈도가 높았음

현재 동작
- 썸네일을 불러오지 못하면 fallback UI가 보인다
- 일부 케이스는 추출 자체가 실패한다

사용자 기대
- 대부분의 릴스는 자동으로 썸네일이 채워져야 한다
- 실패하더라도 사용자가 직접 보완할 수 있어야 한다

문제 성격
- 품질 이슈

영향
- 저장 결과물 완성도가 낮아진다
- 목록에서 시각적으로 다시 찾기 어려워진다
- fallback만으로는 문제를 충분히 해결했다고 느껴지지 않는다

우선순위
- 매우 높음

개선 방향 후보
- 더 나은 외부 썸네일 API 검토
- 자동 추출 성공률 개선
- 실패 시 수동 썸네일 지정 기능 추가
- 수동 지정 방식은 이미지 업로드를 우선 검토

## 다음 우선순위 제안

### 1순위
- 썸네일 자동 추출 문제

이유
- 저장 결과물의 품질 자체를 좌우한다

### 2순위
- 메인 목록 스크롤 위치 복원

이유
- 탐색 UX를 직접 개선한다

## 메모

- 현재까지는 “기능이 없어서”라기보다 “기능은 있는데 실제 사용 경험이 매끄럽지 않은 부분”이 확인된 상태다
- 따라서 다음 단계는 대규모 기능 추가보다, 사용자 테스트 기반 UX 보정 성격이 더 강하다
